初识链表

链表概述

链表是什么呢？简单地说它就是一个由一个个节点组成的一种储存数据的结构。节点里面有着该节点储存的数据和下一个节点的信息（对单项链表而言）。下面我打算用自己实现的简单的单向链表来带着大家熟悉一下诸如插入、删除、检索之类的基本操作。

链表的基本结构

  • 首先是节点，一条链表就是完全由一个个节点链接而成的。
//节点，构成链表的基本元素。
//简单解释一下尖括号中的E，我们可以把它看成一种参数，需要在创建链表对象的时候传入。它就相当于链表中储存的数据的类型。也就是所谓的类型参数。下面所有涉及到节点的操作都需要这个类型参数。
class Node<E> {

    //节点内储存的数据。
    E item;

    //该节点下一个节点的引用。（链表不能像数组一样通过下标来进行访问，想要访问某一个元素只能从头节点开始一个个的往后检索。）
    Node<E> next;

    //节点的构造器，需要传入该节点的数据和下一个节点的信息。
    public Node(E e,Node<E> n) {
        item = e;
        next = n;
    }

}
  • 然后是整体的链表。
//简易的链表，尖括号中的也是类型参数。
public class SimpleLinkedList<E> {

    //Node作为一个静态的内部类存在。 翻译过来也就是说Node类是链表的组成部分，但不依赖于链表的对象存在。
    private static class Node<E> {
        
        E item;

        Node<E> next;

        public Node(E e,Node<E> n) {
            item = e;
            next = n;
        }

    }

    //链表的长度，用一个size来记录。
    int size = 0;  

    //头节点。也就是链表的开始。
    Node<E> head;

}

可以看到，就属性而言，链表中只有记录长度的size和一个头节点head。而从物理层面上来讲，head这个头节点包含了其他所有节点。但是从逻辑角度上来讲，所有节点的地位都是相同的，他们一个连接着一个共同构成了一个链表。我们思考链表的时候可以从逻辑的角度出发或许会更易于理解。

插入操作

插入操作，也就是链表中的add方法。

add有两个重载的方法，一个默认插入在链表的最后方，另一个可以指定插入的位置。

//先是一些辅助方法。

/**
 * 通过链表的节点下标来检索该节点的方法
 * @param index 它是需要的节点在链表中的下标。
 * @return Node<E> 返回该节点本身。
 */>
Node<E> node(int index) {

    //从头结点开始。
    Node<E> x = head;

    //一个一个的向后检索，直到需要的节点。再把它返回回去。
    for ( int i = 0; i < index; i++ ) {
        x = x.next;
    }

    return x;
}

/**
 * 在所给节点之后插入新的节点。
 * @param e 需要插入的新的数据
 * @param pred predecessor也就是所谓的前置节点，他是我们要插入的节点前面的节点。
 */
void linkAfter(E e,Node<E> pred) {
    
    //我们假设pred是一个非空引用。

    //step1:创建新的节点。新的节点应该在pred之后和pred的后置节点之前，所以pred的后置节点就应该是newNode的后置节点。
    Node<E> newNode = new Node<>(e,pred.next);

    //step2:将新的节点连接到原来的pred节点之后。newNode插入之后应该是pred节点的后置节点。
    pred.next = newNode;

}

/**
 * 在头节点之前插入新的节点。
 * @param e 需要插入的元素
 */
void linkHead(E e) {

    Node<E> newNode = new Node<>(e,head);

    head = newNode;

}

/**
 * 在链表的最后插入节点。
 * @param e 需要插入的元素
 */
void linkLast(E e) {

    //插入的节点是在链表的最后方，所以他的后置节点一定是null。
    Node<E> newNode = new Node<>(e,null);

    //考虑一下头指针为空的情况
    if ( head == null) {
        head = newNode;
        return;
    }

}

public void add(int index,E element) {

    //index可能为0，这就需要在头指针之前插入节点。
    if ( index == 0 ) {
        linkHead(e);
    }
    //要在index位置插入一个元素，而我们现在有一个linkAfter方法。所以需要取到index的前置节点。
    Node<E> pred = node(index-1);

    //在pred之后插入新的节点。
    linkAfter(element,pred);

    //加入了一个新的节点，链表的size就需要+1。
    size++;

}

检索操作

检索分为两部分，一是通过节点下标来检索对应节点中的数据。二是通过元素来找到它对应节点的下标。

简单的实现一下get方法也就是第一点通过下标找数据。

/** 
 * 通过下标的检索。
 * @param index 需要查找的下标
 * @return E 返回下标对应的元素
 */
public E get(int index) {
    
    //简便起见，我们假设index是一个合法的下标
    Node<E> n = node(index);

    return n.item;

}
